# 贡献说明

先说结论：欢迎提问题、提修复、提功能，但向本仓库提交代码、文档或其他材料前，请先看完 `docs/legal/CLA.md`。如果你提交了补丁、PR、附件或其他贡献，默认视为你同意 `docs/legal/CLA.md` 里的条款。

## 提交前先确认

- 你有权提交这些内容，不会侵犯他人的版权、专利、商业秘密或合同义务
- 如果你是代表公司、学校、实验室或其他组织提交，已经拿到内部许可
- 你的贡献不包含保密资料、受限 SDK、泄露出来的固件、未授权素材或来路不明的代码
- 如果你引入第三方代码、库、数据或素材，必须同时带上原始许可证说明，并在 PR 里讲清来源

## 你提交贡献后会发生什么

- 你的贡献可以被合并、修改、拒绝、拆分、回退或重写
- 你的贡献会进入本仓库当前使用的 `PolyForm-Noncommercial-1.0.0` 授权体系
- Maintainer 可以按 `docs/legal/CLA.md` 里约定，把你的贡献继续用于后续版本、商业授权或后续换协议

## 建议你在 PR 里写清楚

- 改了什么
- 为什么要改
- 改动影响到哪些板级工程、车型配置或共享模块
- 有没有引入新的第三方依赖
- 你自己做过哪些构建、联调或回归验证

## 对第三方内容的要求

- 优先给链接，不要整包把第三方代码直接塞进来
- 真要引入整包，请把来源、版本、许可证、改动范围一次写清楚
- 带独立许可证的文件，不能删掉人家的版权头和许可证

## 不接受的内容

- 无法确认来源的代码
- 明显带有保密风险或侵权风险的资料
- 只改生成产物、不带源头说明的提交
- 刻意删改第三方版权和许可证信息的提交
